# Binance Orderbook Userscript Architecture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert `scripts/binance-orderbook-trade.user.js` from a 3254-line hand-edited userscript into a modular, tested source tree that still builds to one installable `.user.js` file.

**Architecture:** `src/binance-orderbook-trade/` becomes the development source. `scripts/binance-orderbook-trade.user.js` remains the public installation and update target, but is generated by a build command. Pure business logic is extracted first, DOM behavior second, and side-effect bootstrap last so every migration step can be tested and compared against the generated userscript.

**Tech Stack:** Node.js, npm scripts, `node:test`, `node:assert/strict`, esbuild for non-minified IIFE bundling, and optional jsdom for DOM integration tests after pure unit coverage is established.
